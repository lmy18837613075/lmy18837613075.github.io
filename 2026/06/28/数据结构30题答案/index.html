1. 顺序表与链表的时间复杂度对比

操作 顺序表 链表
随机访问 O(1) O(n)
按值查找 O(n) O(n)
插入/删除（头部） O(n) O(1)
插入/删除（尾部） O(1)* O(n)
插入/删除（中间） O(n) O(n)

*顺序表尾部插入均摊 O(1)

本质区别：

  • 顺序表：连续内存，支持随机访问，但插入删除需移动元素
  • 链表：离散内存，不支持随机访问，但插入删除只需改指针

实现逻辑：

3. 循环队列判满判空（两种方法）

方法1：牺牲一个存储单元

  • 判空：front == rear
  • 判满：(rear + 1) % maxSize == front
  • 入队：rear = (rear + 1) % maxSize
  • 出队：front = (front + 1) % maxSize

方法2：增设 size 变量

  • 判空：size == 0
  • 判满：size == maxSize
  • 入队：rear = (rear + 1) % maxSize; size++
  • 出队：front = (front + 1) % maxSize; size--

方法3：增设 tag 标志

  • 判空：front == rear && tag == 0
  • 判满：front == rear && tag == 1
  • 入队后 tag = 1，出队后 tag = 0

10. AVL 树的四种旋转

LL 型（左左）- 右旋

触发：左子树的左子树插入导致失衡（左高）

操作：A 向右旋转，B 成为新根

RR 型（右右）- 左旋

触发：右子树的右子树插入导致失衡（右高）

操作：A 向左旋转，B 成为新根

LR 型（左右）- 先左后右

触发：左子树的右子树插入

操作：

  1. 对 B 左旋（变成 LL 型）
  2. 对 A 右旋

RL 型（右左）- 先右后左

触发：右子树的左子树插入

操作：

  1. 对 B 右旋（变成 RR 型）
  2. 对 A 左旋

11. 哈夫曼树构造与 WPL 计算

构造过程

  1. 将 n 个权值作为 n 棵只有根节点的二叉树
  2. 选取权值最小的两棵树作为左右子树，构造新二叉树（根节点权值为两者之和）
  3. 从森林中删除这两棵树，加入新树
  4. 重复 2-3，直到只剩一棵树

示例：权值 {5, 3, 2, 4}

WPL 计算

公式：WPL = Σ(叶子权值 × 路径长度)

示例：

  • 权值 5：路径长度 1
  • 权值 4：路径长度 2
  • 权值 2：路径长度 3
  • 权值 3：路径长度 3

WPL = 5×1 + 4×2 + 2×3 + 3×3 = 5 + 8 + 6 + 9 = 28

12. 并查集优化

路径压缩

思想：查找时将路径上的所有节点直接连接到根节点

效果：树高度趋近于 1，查找接近 O(1)

按秩合并

思想：将秩小的树连接到秩大的树下

效果：树高度不超过 O(log n)

综合复杂度

  • 单独路径压缩：均摊 O(log n)
  • 单独按秩合并：O(log n)
  • 两者结合：均摊 O(α(n)) ≈ O(1)（α 是反阿克曼函数）

特性 B 树 B+ 树
关键字存储 所有节点都存 只在叶子节点存
数据记录 所有节点都存 只在叶子节点存
叶子节点链表 无 有（双向链表）
层数 相同关键字数下，B 树更矮 相对更高

存储空间 O(n²) O(n + e)
适用图类型 稠密图 稀疏图

求顶点的度 O(n) O(degree)
遍历所有边 O(n²) O(n + e)

  • 稀疏图（e << n²）：邻接表

20. Dijkstra 算法与负权边

为什么失效？

Dijkstra 的贪心策略：每次选择距离最小的未访问节点，认为已经找到最短路径。

负权边的问题：

  • Dijkstra 从 A 出发：
    • A→B = 1（认为 B 的最短路径是 1）
    • A→C = 2
    • C→D = 3
  • 但实际 A→C→D = 2+1 = 3，而 A→B→D = 1+(-3) = -2 更短！

根本原因：负权边可能导致已确定的最短路径被更新，违反贪心策略的”一旦确定不再改变”原则。

解决方案：使用 Bellman-Ford 算法（可以处理负权边）。

22. 折半查找判定树

构建方法

示例：数组长度 n = 11，下标 0~10

规则：

  • 根节点：mid = (low + high) / 2
  • 左子树：在左半部分递归
  • 右子树：在右半部分递归

查找失败次数

失败节点：叶子节点的左右孩子（外部节点）

计算方法：

  • 查找失败时，最终落在外部节点
  • 失败次数 = 从根到外部节点的路径长度

示例：

  • 查找 4：5→2→3→失败，比较 3 次
  • 查找 11：5→8→9→10→失败，比较 4 次

ASL（平均查找长度）：

  • 成功：ASL = (Σ 每层节点数 × 层数) / n
  • 失败：ASL = (Σ 外部节点路径长度) / (n+1)

27. 希尔排序的增量序列

常见增量序列

1. Shell 增量：h = ⌊n/2⌋, ⌊h/2⌋, ..., 1

  • 例：n=10 → 5, 2, 1
  • 时间复杂度：O(n²)

2. Hibbard 增量：h = 2^k - 1

  • 例：1, 3, 7, 15, 31, …
  • 时间复杂度：O(n^1.5)

3. Knuth 增量：h = (3^k - 1) / 2

  • 例：1, 4, 13, 40, 121, …
  • 时间复杂度：O(n^1.5)

4. Sedgewick 增量：更复杂的序列

  • 时间复杂度：O(n^1.3)

增量选取原则

  1. 最后一个增量必须是 1
  2. 增量之间尽量互质
  3. 增量递减不要太快

影响：

  • 增量序列决定了算法的效率
  • 好的增量序列可以减少比较和移动次数

28. 稳定排序 vs 不稳定排序

本质区别

稳定排序：相等元素的相对顺序在排序后不变

不稳定排序：相等元素的相对顺序可能改变

示例

常见排序算法的稳定性

稳定：

  • 插入排序
  • 冒泡排序
  • 归并排序
  • 基数排序

不稳定：

  • 快速排序
  • 堆排序
  • 选择排序
  • 希尔排序

为什么重要？

  • 多关键字排序：先按次要关键字排序，再按主要关键字排序（需要稳定）
  • 保持原有顺序：某些场景需要保持相等元素的原始顺序

29. 最好/最坏时间复杂度差异

差异最大的排序

快速排序：

  • 最好：O(n log n)（每次划分均匀）
  • 最坏：O(n²)（已排序或逆序）
  • 差异：O(n log n) → O(n²)

冒泡排序：

  • 最好：O(n)（已排序，加标志位）
  • 最坏：O(n²)（逆序）

插入排序：

  • 最好：O(n)（已排序）
  • 最坏：O(n²)（逆序）

为什么快排差异最大？

  • 快排的性能高度依赖于 pivot 的选择
  • 最好情况：每次划分成两半 → 递归树平衡 → O(n log n)
  • 最坏情况：每次划分成 0 和 n-1 → 递归树退化成链表 → O(n²)

改进：随机选择 pivot 或三数取中，可以避免最坏情况。

30. KMP 算法的 next 数组

next 数组的含义

next[j]：当模式串第 j 个字符失配时，应该回退到哪个字符继续比较。

本质：模式串的前 j 个字符组成的子串中，最长相等前后缀的长度。

前后缀定义

  • 前缀：包含首字母，不包含尾字母的所有子串
  • 后缀：包含尾字母，不包含首字母的所有子串

示例：ABAB

  • 前缀：A, AB, ABA
  • 后缀：B, AB, BAB
  • 最长相等前后缀：AB（长度 2）

求 next 数组
