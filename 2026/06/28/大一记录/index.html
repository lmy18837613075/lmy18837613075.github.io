大一复盘：从零基础到做出自己的IoT项目

大一复盘：从零基础到做出自己的IoT项目

2025.9 - 2026.6，一个普通大一学生的折腾一年

写在前面

转眼间大一就结束了。回头看这一年，从刚入学时对编程一知半解，到现在能独立完成一个完整的IoT项目，中间踩了不少坑，也收获了很多。记录一下这一年的成长轨迹，也算是给自己一个交代。

算法竞赛：从入门到坚持

大一上学期加入了学校的ACM竞赛队。说实话，刚开始看到那些算法题的时候，内心是崩溃的——什么贪心、动态规划、图论，听都没听过。

但竞赛队的氛围很好，学长会带着刷题，队友之间互相讨论。从最基础的C++语法开始，到后来能在洛谷上独立解题，这个过程虽然痛苦，但每次AC一道题的成就感是真的爽。

现在主要用C++刷题，偶尔也会用Python写一些小工具。虽然离大佬们还差得远，但至少算是入了门。

IoT项目：从想法到落地

物联网设计大赛：「凉舍智控」

大一下学期，第一个项目是”凉舍智控”——一个宿舍微气候主动调控系统。

为什么做这个项目？ 很简单，因为宿舍夏天真的很热（没有空调），而且每个人的体感温度不一样。我们想用技术手段让宿舍环境更舒适。

技术方案：

  • 硬件：ESP32开发板 + 温湿度传感器 + 红外模块
  • 软件：Arduino编程 + MQTT通信 + 边缘AI推理
  • 亮点：设计了一个SleepComfortNet睡眠舒适度分类模型，只有85KB，断网也能工作

这个项目让我第一次完整体验了从需求分析、方案设计、硬件调试到文档撰写的全流程。虽然还有很多不完善的地方，但能把想法变成可演示的原型，已经很满足了。

学习项目：大语言模型驱动的智能家居系统

除了物联网设计大赛，我还学习其他项目——“大语言模型驱动的智能家居系统”。

项目亮点：

  • 将大语言模型（LLM）与智能家居控制结合，实现更自然的交互方式
  • 通过自然语言指令控制家居设备，降低使用门槛
  • 探索AI在传统IoT场景中的应用可能性

这个项目让我第一次接触到LLM的应用开发，虽然还在准备阶段，但从方案设计到PPT制作、文档撰写，整个过程让我对AI+IoT的融合有了更深的理解。

个人长期项目：智能空间控制系统

比赛之后，我萌生了一个更大的想法：能不能做一个通用的智能家居控制系统？不局限于某个场景，而是能适配宿舍、办公室、整屋等不同空间。

于是开始了”智能空间控制系统”这个项目：

Phase 1（已完成）：

  • ESP32开发环境搭建完成
  • MQTT双向通信调通
  • 完整的设计文档输出（四层架构：感知层→协议抽象层→边缘决策层→执行控制层）

Phase 2（进行中）：

  • FastAPI后端开发
  • 传感器数据采集与处理
  • Web管理界面

这个项目可能会持续很久，但我觉得这种长期迭代的项目才能真正锻炼能力。

技术栈拓展

这一年接触的技术栈比高中多了很多：

  • 嵌入式开发：Arduino → ESP32，学会了MQTT协议、FreeRTOS、传感器数据采集
  • 后端开发：Python + FastAPI，初步理解了API设计、数据库操作
  • 容器化部署：Docker Compose，虽然还不太熟练，但已经能把服务跑起来
  • AI应用：研究Agent开发，尝试搭建本地AI助手

最大的感受是：技术是相通的。学会了一个领域的基础概念，跨领域学习会快很多。

宿舍生活：在有限条件下创造舒适

宿舍条件一般，没有空调，夏天只能靠风扇续命。但我觉得，环境是可以改造的。

花了些时间研究上铺空间的利用，规划了一个”太空舱”式的改造方案：床帘、遮光布、小灯、置物架……虽然预算有限，但能住得更舒服一点，学习效率也会高一些。

这也是一种能力吧——在有限条件下，找到最优解。

一些感悟

  1. 做项目比刷题更重要：算法竞赛很重要，但真正让你理解技术的是完整的项目经历
  2. 不要怕从零开始：大一的时候我对嵌入式一窍不通，但现在也能独立调试硬件了
  3. 记录很重要：写博客、写文档，不仅是给别人看，更是给自己梳理思路
  4. 找到同路人：无论是竞赛队还是开源社区，有人一起走会走得更远

大二展望

接下来的一年，几个主要方向：

  • 继续算法竞赛训练，争取在CCPC等比赛中拿到更好的成绩
  • 推进智能空间控制系统的开发，争取能实际应用
  • 深入学习Agent/AI应用开发，这是未来的趋势
  • 开始准备考研，英语是重点突破方向

大一这一年，算是从”学生”向”开发者”转变的开始。路还很长，但至少方向是对的。
